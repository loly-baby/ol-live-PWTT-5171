# Stamp Reuse MVP 产品需求文档（单次付费 + 轻会员版）

## 1. 产品定义

### 产品名称
Stamp Reuse MVP

### 一句话定位
把现实中的章，快速变成可复用的透明底数字章，并秒盖到任意文件上。

### 英文定位
Turn a real-world stamp into a reusable transparent digital stamp, then place it on any file in seconds.

---

## 2. 产品核心判断

普通“做章工具”已经很多，但用户真正愿意为之付费的，不只是“做出一个章”，而是：

1. 把已有的真实印章数字化
2. 变成透明底资产
3. 继续盖到新文件上
4. 直接导出可发送、可提交、可归档的文件

所以本产品不是简单的 Seal Generator，而是：

**Document Stamping + Stamp Extraction**

同时，产品不适合一上来做复杂多层订阅体系，更合理的路径是：

**先保留单次付费，再增加一个轻会员。**

---

## 3. 核心用户

### 3.1 小商家 / 跨境卖家
- 已经在线下或纸质流程里有“自己的章”
- 需要把章快速盖到 invoice、packing list、售后单据上

### 3.2 轻办公用户
- 财务、行政、助理、运营
- 需要在 PDF 或图片文件上放已批准 / 已收款 / 已接收等标识章

### 3.3 高频复用用户
- 每周 / 每月都要重复提取、保存、盖章、导出
- 不适合每次都单独购买，更适合轻会员

---

## 4. MVP 核心价值

### 模块 A：文件盖章
- 上传 PDF / 图片
- 拖拽章到任意位置
- 调整大小、透明度、旋转角度
- 导出新文件

### 模块 B：旧章提取
- 上传手机拍摄的盖章文件照片
- 自动识别章主体颜色（红 / 蓝）
- 自动去纸张底色，输出透明底 PNG
- 保存为“我的章”并继续复用

### 模块 C：新章生成
- 从模板快速生成可用数字章
- 作为提取能力之外的补充入口

### 模块 D：轻会员
- 为高频用户提供一个月费方案
- 保留单次付费，同时给重复使用者一个更自然的升级路径

---

## 5. MVP 必须交付的用户流程

### 流程 1：生成新章并盖章
1. 进入 workspace
2. 选择模板并编辑文字
3. 保存到章库
4. 上传一份 PDF 或图片
5. 把章拖到目标位置
6. 导出新文件

### 流程 2：提取旧章并复用
1. 上传手机拍照文件
2. 选择红章 / 蓝章 / 自动识别
3. 调整阈值与清理强度
4. 获得透明底结果
5. 保存为章资产
6. 上传新文件
7. 拖拽章到新文件并导出

### 流程 3：轻会员升级
1. 用户完成过一次或几次单次工作流
2. 在 pricing 页看到 Pro 方案
3. 输入邮箱并进入 Stripe Checkout
4. 成功后在 billing 页查看状态
5. 后续通过 Stripe portal 管理续费或取消

---

## 6. 正确的产品承诺

不能承诺：
- 任意手机照片都 100% 完美一键抠章

应承诺：
- 大多数清晰照片可自动提取
- 复杂情况可快速微调

一句话表达为：

**自动提取 + 手动微调 + 一键复用**

---

## 7. 功能范围

### 7.1 第一阶段必须有
- 新章模板生成
- 旧章提取
- 当前会话章库
- PDF / 图片文件上传
- 拖拽放置章
- 调整大小 / 旋转 / 透明度
- 导出图片 / PDF
- 单次付费 checkout
- 轻会员 checkout
- billing 页会员查询
- Stripe portal 入口

### 7.2 第二阶段增强
- 透视校正
- 画笔擦除杂点
- 多章自动检测
- 完整账号系统
- 团队章库
- 权限分层与用量限制
- one-time vs Pro 转化分析

## 8. 收费方案

### 单次付费
- 单文件盖章导出：$2.9
- 单次旧章提取并保存：$4.9
- 提取 + 盖章打包：$6.9

### 轻会员
- Pro：$12.9 / 月

### Pro 包含
- 无限高清导出
- 无限旧章提取与保存
- 持久化章库
- PDF 盖章包含在内
- Stripe portal 自助管理订阅

### 定价原则
**先卖结果，再卖高频使用权。**

## 9. 技术方案

### 前端
- Next.js App Router
- Tailwind CSS
- react-pdf + pdfjs-dist

### 后端
- Next.js Route Handlers
- Sharp（图片提取、图片合成）
- pdf-lib（PDF 盖章写回）
- Prisma + PostgreSQL（资产 / 订单 / 会员 / 项目）
- Stripe Checkout / Billing Portal / Webhook

## 11. 成功指标

第一轮验证看五个指标：

1. 有多少用户成功提取出第一枚透明底章
2. 有多少用户完成一次“把章盖到文件上”的动作
3. 有多少用户愿意为单次结果付费
4. 有多少单次付费用户愿意升级到 Pro
5. Pro 用户的留存情况如何

---

## 12. 最终判断标准

如果用户持续使用“旧章提取 + 复用盖章”，说明产品不是普通工具，而是真正的办公效率产品。

如果单次付费成立，而高频用户愿意转向 Pro，说明“单次付费 + 轻会员并存”的结构是正确的。
